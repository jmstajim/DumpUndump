Output Contract (VERY IMPORTANT)

You MUST produce a file named dump.txt
• The response must be delivered as a download link to dump.txt.

If you cannot finish everything in one pass (time/token complexity), you MUST:

⸻

Received Dump File Rules

• The dump you receive contains FULL file contents. There is no truncation anywhere.
• Any occurrence of "..." / "…" in the dump you receive is part of the real content, not a marker.

⸻

File Section Rules
• File sections ONLY inside dump.txt. No extra text in dump.txt.
• Each modified or new file must be included with its FULL contents (no diffs).
• No sections for unchanged files.
• If there are no changes, create an empty dump.txt (0 bytes).
• Deletions: to delete a file, output its section with an empty code body.
• Paths must be strictly relative, with:
• no leading /
• no ..
• Number sections consecutively <<<FILE #0001>>> ... <<<END FILE #0001>>> (any width ok, but must match).

⸻

Encoding / Newlines (strict)
• UTF-8 without BOM
• LF line endings
• Unicode normalization NFC
• Every file ends with exactly one real LF (U+000A)
• Do not convert literal escape sequences:
• keep \n, \r, \t, \\ as two-character sequences (backslash + letter)
• Normalize only real newlines (CRLF/CR → LF), but do not touch literal \n text.

⸻

Code Fence Rules
• Default fence: triple backticks.
• If file content contains ``` inside, fence with ~~~ (preferred) or four backticks.

⸻

Workflow You Should Follow
1. Parse the dump and build an internal map: path -> original contents.
2. Read README_LLM/TODO_LLM if present (per invariants).
3. Implement TASK.
4. Update TODO_LLM with your thoughts.
5. Validate before output:
    • only changed/new/deleted files included
    • all paths safe and relative
    • every file ends with LF
    • no accidental escaping/unescaping
6. Output dump.txt as a downloadable file link.

⸻

Chat Response Rules
• Normally: return only the download link.
• If partial: return the download link plus the short “CONTINUE REQUEST” message (≤10 lines), ending with “Reply CONTINUE to proceed.”
